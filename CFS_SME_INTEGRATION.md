# 쿠팡 CFS(쿠펀치) → SME 일자리 통합 전략

> **포지션**: 「당근알바급 발견(지도·알림핀) + 쿠팡급 운영(셔틀·스케줄·출근)」 하이브리드  
> **파일럿**: 다이소 세종 물류센터(채용대행) — 셔틀·탑승시간 안내 중심, 실시간 버스 추적은 Phase 2

---

## 1. 쿠팡 내부 모델 요약 (CFS / Coupunch)

대규모 FC(풀필먼트센터)에서 **단기·계약·일용** 인력을 수만 명 규모로 돌리기 위한 **내부 전용** 스택.

| 레이어 | 제품(대외명) | 역할 |
|--------|-------------|------|
| 작업자 앱 | **쿠펀치(Coupunch)** | 가입, 스케줄 신청, 공고/확정, 출퇴근, 급여·공지 |
| 통근 앱 | 헬로버스·크루버스·쿠팡셔틀 등 | 탑승권/QR, 노선·시간표, (선택) 실시간 위치 |
| 운영 | FC 내부 WMS/인력 시스템 | 배정, 출근율, no-show, 인센티브, 식사·셔틀 복지 홍보 |

### 쿠펀치 — 작업자 여정

1. **가입** — 이름·생년·연락처·주소(셔틀 매칭용)
2. **스케줄 선택** — 센터·조(주/오후/심야)·날짜 직접 신청
3. **지원·확정** — 실시간 공고 → 신청 → 푸시 확정
4. **출퇴근** — geofencing / QR / 앱 인증
5. **셔틀** — 노선·시간 확인, 탑승권(별도 셔틀 앱 연동)
6. **유지** — 급여·인센티브·식사·친구초대·공지 푸시

### 셔틀 운영 — 핵심 복지

- 센터별 **노선·시간표** 웹/앱 공개
- **탑승권·QR** → 부정승차 방지
- (고도화) **실시간 버스 위치·도착 예정** 푸시
- 교외 FC에서 **지원·유지율**을 끌어올리는 1순위 수단

### 쿠팡이 증명한 것

- 교통 불편 지역도 **셔틀 + 앱 내 일원화**면 인력 풀 확보 가능
- 공고 카피: `#급여 #식사·셔틀 무료 #초보OK #인센티브` → 전환 명확
- **플랫폼 밖 연락 차단** + 앱 내 lifecycle → 아웃소싱/브로커 우회 어려움

## 2. SME(중소 창고·물류·제조)로 다듬을 때

쿠팡 전체를 복제하지 않고 **축소·간소화**한 패키지.

| 쿠팡 (대기업 FC) | 일자리 SME 팩 |
|------------------|---------------|
| 전국 1000대 버스 fleet 관리 | **노선·시간표 등록** (엑셀/폼), 탑승 정원 수동 |
| 실시간 GPS 전 차량 | **MVP: 시간표 + 탑승 예약 + 30분 전 알림** (Phase 2: 위치) |
| 내부 HR 대시보드 | **기업 앱 + `1jari-admin`** (승인·블랙리스트·정산) |
| 무료 셔틀·식사 대규모 복지 | **「셔틀 있음」지도 필터 + 상세 시간표** 강조 |

### 다이소 세종(채용대행) 맥락

- 고객사(다이소)에게 **「내부 시스템 써보세요」** 는 거부감 → **우리가 지원자만 많이 채워주면 됨** (재직 기간 수수료)
- **필수**: 셔틀 **탑승 시간·정류장** 안내
- **불필요(MVP)**: 실시간 버스 추적
- **유지**: 지도 **「셔틀 있음」** 필터 → 셔틀 유무로 지원 몰림 유도

## 3. 일자리 알림핀 + CFS 결합 — 시너지

```
[외부 유입: 당근·현수막·채용대행]
        ↓
[앱 가입 · 주소 등록]
        ↓
[지도 알림핀 발견] ←── 푸시 패키지(1km·거점)
        ↓
[공고 상세: 셔틀 시간표·노선 지도]
        ↓
[1클릭 지원 · 교대 선택 · 셔틀 정류장 예약]
        ↓
[내 지원 / 근무 · 체크인 · (Phase2) 탑승 QR]
        ↓
[재직/출근 확인 → 수수료]
```

| 레이어 | 역할 | 경쟁 대비 |
|--------|------|-----------|
| **알림핀·푸시** | 발견·유입 | 당근/알바몬 지도 검색과 동급 **+ 위치 타겟 푸시** |
| **셔틀·스케줄** | 전환·유지 | 쿠팡 내부 수준을 **SME 가격**으로 |
| **출근·수수료** | 수익·신뢰 | no-show 감소, 성과 기반 과금 |

### 결론: **알림핀 유지 + CFS 워크플로우 심화** (권장)

| 방향 | 장점 | 리스크 |
|------|------|--------|
| **결합 (권장)** | 유입(핀) + 전환(셔틀) 동시 | 개발 범위 넓음 → 단계적 MVP |
| 핀만 | 푸시 수익 단순 | 교외 창고 **셔틀·스케줄** 약함 → 이탈 |
| CFS만 | 물류 타겟 이해도 빠름 | **지도 발견** 약함 → 초기 성장 느림 |

**근거**: 알바몬·당근은 지도가 핵심 UX. 쿠팡은 셔틀+앱 lifecycle로 유지. Wonolo는 위치+교통 정보 결합.

## 5. 통합 UX 흐름 (구현 기준)

### 구직자

1. **지도** — 핀·클러스터·`전체` / `셔틀 있음` 필터
2. **공고 상세** — 시급·교대·`ShuttleTimetableTable`·노선 지도 오버레이
3. **지원** — `JobApplyFlowSheet`: 날짜/조 → 셔틀 정류장·시간 → (선택) 셔틀 없이 지원
4. **내 지원** — 확정·셔틀 예약·리마인더 배너
5. **근무** — `ShiftCheckInPage` GPS / QR 코드
6. **채팅** — 앱 내 연락 (오프플랫폼 TOS)

### SME 구인자

1. **공고** — 무료 등록 + 알림핀·푸시 패키지
2. **셔틀** — `ShuttleRouteListPage` 노선·정류장·시간
3. **지원자** — 승인·교대 확정·채팅
4. **셔틀·근태 허브** — `CorporateShuttleAttendanceHubPage`
5. **정산** — 일용 출근 수수료 / 상시 재직 수수료 (구직자 UI에는 미노출)

### 플랫폼 Admin (`1jari-admin`)

- 기업·공고·푸시·결제·내부 최적화(연령·성별·셔틀 AI) — 채용대행·블랙리스트 확장 예정

## 7. MVP 우선순위 (다이소 세종)

1. ✅ 지도 핀 + **셔틀 있음** 필터 + 시간표 상세  
2. ✅ 지원 시 **셔틀 정류장·탑승 시간** 선택  
3. ✅ 내 지원 / 근무에서 **셔틀·출근** 한 화면  
4. 🔶 Admin **수동 승인** (웹 + 기업 앱)  
5. ⬜ 당근 공고 → 앱 딥링크 UTM  
6. ⬜ 파일럿 KPI: 지원율·셔틀 선택률·출근율

## 8. 법·운영 주의

- 플랫폼 **중개** 역할 명시 (직접 고용 아님)
- 구직자 UI에 **수수료·5.5%** 노출 금지 (구인자/Admin만)
- 성별·연령 **내부 최적화**만 — 공고 문구 차별 금지
- 채용대행: **지원자 풀 공급** 포지션, 고객사 내부 시스템 대체 홍보 지양
